# Risk Backlog

## Purpose
This file tracks deferred bug, security, and optimization findings that were discovered during audits but are not yet approved for implementation. Items are ordered by likelihood of causing extension-visible bugs, with the highest-risk items first.

## Latest Audit Snapshot
- Date: 2026-05-11.
- Scope: popup runtime, Worker endpoints, manifest permissions, locale JSON files, storage/cache behavior, CORS posture, request-budget risks, and DOM injection patterns.
- Static checks passed in the latest recorded validation set: Worker syntax, popup syntax, smoke-test script syntax, manifest JSON, English locale JSON, Turkish locale JSON.
- Latest v1.1 checks passed popup syntax, Worker syntax, manifest JSON, all seven locale JSON files, 71-key locale parity, DOM injection/eval-like scan, zip-root manifest verification, and `tools/smoke-test.js` endpoint shape validation.
- Security scan found no `innerHTML`, `outerHTML`, `insertAdjacentHTML`, `eval`, `new Function`, or `document.write` usage.
- Manifest still has no broad Chrome API permissions and only uses the Worker host permission.
- Deployed Worker now rejects null/web/raw no-Origin requests; published and unpacked-style Chrome extension origin smoke tests return `200`, and browser-like no-Origin Chrome extension fetches return `200`.

## Deferred Findings

| Priority | Area | Finding | Impact | Proposed Fix | Status |
|---:|---|---|---|---|---|
| 1 | Popup lazy data | Standings and match-detail error states can remain cached for the whole popup session after one failed request. | A temporary ESPN/Worker/network failure can make a league or match detail look broken until the popup is reopened. | Add short error TTLs, retry-on-reopen/retry-on-section-open behavior, or a small retry action. | Implemented in v1.0.5 with 60s lazy error retry TTL |
| 2 | Popup storage | Some `localStorage` reads/writes for power and favorites are not fully wrapped in safe helpers. | Storage quota/private-mode/blocked-storage edge cases can break popup startup or interactions, including tiny-popup-like failures. | Centralize all storage access through safe get/set/remove helpers with try/catch and sane fallbacks. | Implemented in v1.0.5 with safe storage helpers |
| 3 | Popup request guard | The daily request counter is consumed before a request completes. | Aborted, timed-out, or failed requests can still burn the user's local 2,000/day budget and make the app pause earlier than needed. | Track attempted and successful requests separately, or only count completed backend responses while keeping a smaller failed-request safety cap. | Implemented in v1.0.7 with separate `attempted` and `successful` counters; the limit uses successful JSON responses |
| 4 | Popup stale UX | Visible header status text has been removed, while internal status states still include pause/error/limit states. | Users may see stale data without an obvious reason when auto-pause or inactivity pause kicks in. | Add a subtle non-intrusive stale/paused indicator in the content area or footer, only when action is needed. | Partially implemented in v1.0.6: first-load request-limit state now shows a visible request-limit error card |
| 5 | QA workflow | Local `file://` preview is intentionally CORS-blocked after published-origin hardening. | Layout preview can be confused with real runtime behavior, causing false bug reports about missing live data. | Keep using installed extension runtime for live QA; optionally add a dev-only mock-data mode. | Implemented in v1.0.6 with a dedicated file-preview/runtime-required message |
| 6 | Worker abuse control | CORS is restricted to Chrome extension origins and browser-like no-Origin extension fetches, but server-to-server clients can spoof allowed-looking request headers. | Direct abuse can still consume Cloudflare Free plan request quota even though normal websites, `file://` previews, and raw no-Origin clients are blocked. | Add Cloudflare WAF/rate limiting; optionally add a backend-side request guard if traffic grows. | Partially implemented in Worker: null/web/raw no-Origin requests return 403, extension-compatible no-Origin fetches are allowed, and fresh cache-miss CORS was fixed; Cloudflare WAF/rate limiting still recommended |
| 7 | Backend quota | There is no backend-side rate limit beyond cache, validation, and CORS. | Client-side 2,000/day guard can be bypassed by direct Worker calls or modified clients. | Add Cloudflare dashboard rate limiting/WAF rules before traffic grows; consider KV/Durable Object guards only if needed. | Decision pending |
| 8 | Remote logos | League/team images depend on ESPN/Wikimedia/official CDN availability and styling contrast. | Broken, white-on-white, tiny, or outdated logos are visible UX bugs and can affect store screenshots. | Bundle key league logos locally where licensing allows; keep remote team logos with fallbacks/overrides and disclose remote images in privacy/store copy. | Partially implemented in v1.0.7: all 28 curated league logos are packaged locally; Cancún FC has a Worker-side official-source override; most team logos remain remote ESPN assets |
| 9 | ESPN unofficial API | ESPN response shapes and coverage can change without notice. | Normalization, standings, or detail sections can become empty or partially broken. | Add lightweight schema guards, a smoke-test script, and periodic endpoint QA for key leagues. | Partially implemented in v1.0.7 with `tools/smoke-test.js`; deeper schema guards/automated scheduling remain future work |
| 10 | Extra scoreboard probes | UEFA Conference League uses an extra direct scoreboard probe on live cache misses. | Small subrequest increase on Worker cache miss; mostly budget-related, not a user-facing bug. | Keep only while useful; make extra probes configurable or season-aware if request budget tightens. | Decision pending |
| 11 | Team logo fallback hygiene | Worker constructs fallback ESPN team/league logo URLs from upstream IDs. | Low-risk because source is ESPN data and it is image-only, but malformed IDs could create odd image URLs. | Sanitize fallback team and league logo IDs to a safe numeric/slug pattern before building the URL. | Implemented in v1.1 with `sanitizeEspnAssetId()` |

## Notes
- These are not current blockers for the published extension.
- Safe storage helpers and lazy error retry TTLs were implemented in extension `1.0.5`; file-preview and request-limit visible messaging were improved in extension `1.0.6`; request guard semantics, packaged league logos, and smoke-test tooling were added in extension `1.0.7`.
- Worker raw no-Origin rejection was deployed in Worker version `316ebac3-6762-4214-9c6a-7f83b5e08ac1`; fresh cache-miss CORS handling for Chrome extension origins was deployed in Worker version `5a8886ee-e38d-4ed0-a494-33976a76f32f`; extension-compatible browser-like no-Origin fetch support was deployed in Worker version `e86cbb85-cc11-4dbe-98d8-38dc9df341d9`; ESPN logo asset-ID sanitization was deployed in Worker version `5ca9c033-f559-4f80-914f-602fa05fd230`; the Cancún FC team-logo override was deployed in Worker version `e145b8fb-f760-45ac-b0ad-7c0986b59159`.
- The highest-impact remaining quota/security fix is still Cloudflare WAF/rate limiting, because server-to-server clients can spoof allowed-looking request headers.
- Git push is currently blocked by an incorrect/inaccessible remote (`https://github.com/Atakan0zkan/aozka.git` returned `Repository not found`); this is a release-process issue, not an extension runtime blocker.
- Any implementation should be followed by static validation, real installed-extension QA, and store zip regeneration if extension runtime files change.
